## 🚨 Root Cause: **Nginx fails because `upstream "app"` is not found**

From your logs:

```log
[emerg] host not found in upstream "app" in /etc/nginx/conf.d/default.conf:14
```

This means:
> Nginx is trying to proxy requests to a service called `app`, but **no such container is running**, or the **network/DNS resolution failed**.

But wait — your `docker service ls` shows:

```
laravel_app          replicated   0/2        instasure-dockerized_app:latest
```

👉 **0/2 = no containers are running.**

So the real problem is: **Your Laravel app container is not starting at all.**

And since the app isn't running → Nginx can't find `app` as upstream → Nginx crashes → everything fails.

---

## 🔍 Step-by-Step Diagnosis

### ✅ 1. MySQL is Starting (Good!)
From `docker service logs laravel_mysql`:
- Database initialized
- `laravel_db` created
- MySQL ready for connections

✅ So **MySQL is OK**.

---

### ❌ 2. Laravel App Container Is Not Running
You ran:

```bash
docker service logs laravel_app
```

And got **no output**.

That means:
- The container **never started**, or
- It started and crashed immediately (so fast that no logs were printed)

To see **why it didn’t start**, use:

```bash
docker service ps laravel_app --no-trunc
```

This will show you the **exact error**.

---

### 🔥 Most Likely Cause: Image Not Built or Missing Entrypoint

Your compose file uses:

```yaml
image: instasure-dockerized_app:latest
```

But did you **build** this image?

Run:

```bash
docker images | grep instasure-dockerized_app
```

If **no output**, then the image doesn’t exist → containers can’t start.

## ✅ How to Fix This: Step-by-Step

### ✅ Step 1: Build Your Laravel App Image

Go to your project root (where `Dockerfile` is):

```bash
cd ~/gitlab_project/instasure-dockerized
```

Then build:

```bash
docker build -t instasure-dockerized_app:latest .
```

Make sure your `Dockerfile` exists and has something like:

```Dockerfile
FROM php:8.2-fpm

WORKDIR /var/www/html

COPY . .
RUN composer install --optimize-autoloader --no-dev

EXPOSE 9000
CMD ["php-fpm"]
```

> ⚠️ If you're using Apache/Nginx inside the app container, adjust accordingly.

---

### ✅ Step 2: Verify the Image Exists

```bash
docker images | grep instasure-dockerized_app
```

Should show:

```
instasure-dockerized_app   latest   abc123def456   5 seconds ago   250MB
```

---

### ✅ Step 3: Redeploy the Stack

```bash
docker stack rm laravel
docker stack deploy -c docker-stack.yml laravel
```

Wait 10–30 seconds, then check:

```bash
docker service ls
```

Now you should see:

```
laravel_app     replicated   2/2        instasure-dockerized_app:latest
laravel_nginx   replicated   2/2        nginx:alpine
...
```

✅ Both app and nginx running.

---

### ✅ Step 4: Check Nginx Config

Your Nginx config binds:

```nginx
upstream app {
    server app:9000;   # ← This must match the service name in docker-compose
}
```

In your `docker-stack.yml`, the Laravel service is named:

```yaml
services:
  app:   # ← This becomes the DNS name
    ...
```

So `upstream app:9000` is correct ✅

Also ensure both `app` and `nginx` are on the same network:

```yaml
networks:
  - laravel_network
```

Which they are ✅
