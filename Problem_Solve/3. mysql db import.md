# Docker MySQL Database Import Tutorial

This tutorial covers multiple methods to import SQL database files into a MySQL container running in Docker.

## Prerequisites

- Docker and Docker Compose installed
- Running MySQL container
- SQL dump file ready for import
- Basic knowledge of MySQL credentials

## Method 1: Copy File to Container (Quick & Simple)

### Step 1: Copy SQL file to running container
```bash
# Syntax: docker cp <source_file> <container_id>:<destination_path>
docker cp ~/path/to/your/database.sql <container_id>:/tmp/database.sql

# Example:
docker cp ~/gitlab_project/instasure-dockerized/DB/laravel_db.sql a03e73776eab:/tmp/laravel_db.sql
```

### Step 2: Import from inside the container
```bash
# Enter the container
docker exec -it <container_id> bash

# Import the database (will prompt for password)
mysql -u username -p database_name < /tmp/database.sql
```

**Pros:** Quick and easy for one-time imports  
**Cons:** File only exists temporarily in container

## Method 2: Volume Mount (Recommended for Development)

### Step 1: Modify docker-compose.yml
```yaml
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_DATABASE: your_database
      MYSQL_USER: your_user
      MYSQL_PASSWORD: your_password
      MYSQL_ROOT_PASSWORD: root_password
    volumes:
      - mysql_data:/var/lib/mysql
      - ./DB:/docker-db  # Mount your DB folder to container
    ports:
      - "3306:3306"

volumes:
  mysql_data:
```

### Step 2: Restart containers
```bash
docker-compose down
docker-compose up -d
```

### Step 3: Import the database
```bash
# Enter container
docker exec -it <container_name> bash

# Import from mounted volume
mysql -u username -p database_name < /docker-db/database.sql
```

**Pros:** Persistent access to DB files, great for development  
**Cons:** Requires container restart

## Method 3: Direct Import from Host

### Option A: With Password Prompt
```bash
# From your project directory
docker exec -i <container_id> mysql -u username -p database_name < ./path/to/database.sql
```

### Option B: Inline Password (Less Secure)
```bash
# Replace 'password' with actual password
docker exec -i <container_id> mysql -u username -ppassword database_name < ./path/to/database.sql
```

### Option C: Using Environment Variable
```bash
# Set password as environment variable
docker exec -i -e MYSQL_PWD=your_password <container_id> mysql -u username database_name < ./path/to/database.sql
```

**Pros:** No need to enter container  
**Cons:** File path must be accessible from host

---

## Method 4: Using Docker Compose Exec

```bash
# From your project directory
docker-compose exec mysql bash -c "mysql -u username -p database_name < /path/to/mounted/file.sql"

# Or with environment variable
docker-compose exec -e MYSQL_PWD=password mysql mysql -u username database_name < /path/to/file.sql
```

**Pros:** Uses service names instead of container IDs  
**Cons:** Requires docker-compose

## Method 5: Init Scripts (For New Containers)

### Step 1: Create init directory structure
```
project/
├── docker-compose.yml
├── mysql-init/
│   └── 01-import.sql
└── DB/
    └── database.sql
```

### Step 2: Modify docker-compose.yml
```yaml
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_DATABASE: your_database
      MYSQL_USER: your_user
      MYSQL_PASSWORD: your_password
      MYSQL_ROOT_PASSWORD: root_password
    volumes:
      - mysql_data:/var/lib/mysql
      - ./mysql-init:/docker-entrypoint-initdb.d
      - ./DB:/docker-db
    ports:
      - "3306:3306"
```

### Step 3: Create init script
```sql
-- mysql-init/01-import.sql
USE your_database;
SOURCE /docker-db/database.sql;
```

**Pros:** Automatic import on container creation  
**Cons:** Only runs on fresh containers

## Common Issues & Solutions

### Issue 1: Password Authentication Errors

**Problem:** `ERROR 1045 (28000): Access denied`

**Solutions:**
1. Check password in docker-compose.yml
2. Use correct password syntax:
   - Interactive: `-p` (will prompt)
   - Inline: `-pPASSWORD` (no space)
   - Environment: `MYSQL_PWD=password`

### Issue 2: File Not Found

**Problem:** `No such file or directory`

**Solutions:**
1. Verify file exists: `ls -la /path/to/file`
2. Check if running command from correct location
3. Use absolute paths instead of relative paths

### Issue 3: Permission Denied

**Problem:** Cannot access mounted volumes

**Solutions:**
```bash
# Fix file permissions
chmod 644 /path/to/database.sql

# Fix directory permissions  
chmod 755 /path/to/DB/

# Change ownership if needed
sudo chown -R $USER:$USER /path/to/project/
```

### Issue 4: Large File Import Timeouts

**Solutions:**
1. Increase MySQL timeout settings in docker-compose.yml:
```yaml
command: --max_allowed_packet=1073741824 --innodb_log_file_size=256M
```

2. Use `mysql` with specific options:
```bash
mysql -u username -p --max_allowed_packet=1073741824 database_name < large_file.sql
```

## Best Practices

1. **For Development:** Use Method 2 (Volume Mount) for easy access to DB files
2. **For Production:** Use Method 5 (Init Scripts) for automated setup
3. **For Quick Tasks:** Use Method 1 (Copy File) for one-time imports
4. **Always backup** existing data before importing
5. **Use strong passwords** and avoid inline password methods in production
6. **Test imports** on small datasets first

---

## Example Complete Workflow

```bash
# 1. Check running containers
docker ps

# 2. Copy database file
docker cp ./DB/backup.sql mysql_container:/tmp/backup.sql

# 3. Create database if it doesn't exist
docker exec -it mysql_container mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS myapp_db;"

# 4. Import the database
docker exec -i mysql_container mysql -u myapp_user -p myapp_db < /tmp/backup.sql

# 5. Verify import
docker exec -it mysql_container mysql -u myapp_user -p -e "USE myapp_db; SHOW TABLES;"
```

## Troubleshooting Checklist

- [ ] Container is running (`docker ps`)
- [ ] Database exists or is created
- [ ] User has proper permissions
- [ ] File path is correct and accessible
- [ ] File format is valid SQL
- [ ] Sufficient disk space available
- [ ] Network connectivity between host and container
- [ ] MySQL service is ready (not still initializing)

Remember: Always test your import process in a development environment before applying to production!
