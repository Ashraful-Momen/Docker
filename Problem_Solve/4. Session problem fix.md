The issue you're experiencing is related to cookie security policies when accessing your Laravel app from different devices. The browser console error gives us a clear clue: cookies with `SameSite=None` require the `Secure` attribute, which only works over HTTPS.

## Root Cause Analysis

1. **Cookie Security Conflict**: You have `SESSION_SECURE_COOKIES=true` but are using HTTP (`APP_URL=http://36.255.69.72:8080`)
2. **Cross-Origin Cookie Policy**: When accessing from other devices, browsers enforce stricter cookie policies
3. **CSRF Token Issues**: Error 419 indicates CSRF token mismatch, often caused by session cookie problems

## Solution

Here's how to fix this for your development environment:

### 1. Update your `.env` file:

```env
# Change these settings
SESSION_SECURE_COOKIES=false
SESSION_SAME_SITE=lax
SESSION_DOMAIN=.36.255.69.72
SESSION_HTTPONLY=true

# Keep your existing session config
SESSION_DRIVER=redis
SESSION_LIFETIME=120
```

### 2. Add session configuration to `config/session.php`:

Make sure these values are properly set:

```php
'secure' => env('SESSION_SECURE_COOKIES', false),
'http_only' => env('SESSION_HTTPONLY', true),
'same_site' => env('SESSION_SAME_SITE', 'lax'),
'domain' => env('SESSION_DOMAIN', null),
```

### 3. Update your CORS configuration:

In `config/cors.php`, ensure you have:

```php
'paths' => ['api/*', 'sanctum/csrf-cookie', 'web/*'],
'allowed_methods' => ['*'],
'allowed_origins' => ['*'], // For development only
'allowed_origins_patterns' => [],
'allowed_headers' => ['*'],
'exposed_headers' => [],
'max_age' => 0,
'supports_credentials' => true,
```

### 4. Clear caches and restart:

```bash
docker exec -it laravel_app php artisan config:clear
docker exec -it laravel_app php artisan cache:clear
docker exec -it laravel_app php artisan view:clear
docker-compose restart nginx app
```

### 5. Fix the route caching issue:

Before you can cache routes, fix the duplicate route name error:

```bash
# Find duplicate route names
docker exec -it laravel_app php artisan route:list --name=api_car_insurance_premium_calculation
```

You have duplicate route names. Check your route files and ensure each route has a unique name.

## Alternative Solution (Production-Ready)

For a production environment, you should use HTTPS:

1. Set up SSL certificates (Let's Encrypt, CloudFlare, etc.)
2. Update `APP_URL` to `https://your-domain.com`
3. Keep `SESSION_SECURE_COOKIES=true`
4. Set `SESSION_SAME_SITE=none` only if you need cross-site requests

## Quick Test

After making these changes, test from another device:

```bash
# Check if sessions are working
docker exec -it laravel_app php artisan tinker --execute="session()->put('test', 'working'); echo session()->get('test');"
```

The key insight is that for HTTP connections (development), you need `SESSION_SECURE_COOKIES=false` and `SESSION_SAME_SITE=lax`. For HTTPS (production), you can use secure cookies with `SameSite=None`.
