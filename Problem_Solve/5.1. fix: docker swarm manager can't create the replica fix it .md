## 🚨 Root Cause: **Placement Constraints Are Blocking Container Start**

Look at your `docker-stack.yml`:

```yaml
placement:
  constraints:
    - node.role == worker
```

This means:
> "Only run this service on nodes with role **worker**"

But in a **single-node Swarm** (which you likely have), your node is probably a **manager**, not a worker.

So Docker says:
> "I can't run `laravel_app` here — this node is a manager, but the constraint requires a worker."

Hence: ❌ **Pending forever**

---

## ✅ Solution: Fix Placement Constraints

You have **two options**:

---

### ✅ Option 1: Remove Placement Constraint (Recommended for Single Node)

Edit your `docker-stack.yml` → **Remove or comment out** the placement block:

```yaml
app:
  image: instasure-dockerized_app:latest
  deploy:
    replicas: 2
    restart_policy:
      condition: on-failure
      delay: 5s
      max_attempts: 3
      window: 120s
    update_config:
      parallelism: 1
      delay: 10s
      failure_action: rollback
    # Remove this block:
    # placement:
    #   constraints:
    #     - node.role == worker
  volumes:
    - app_/var/www/html
  networks:
    - laravel_network
  depends_on:
    - mysql
    - redis
```

> 🔁 Do the same for `nginx`, `mysql`, etc., if they also have `node.role == worker`

---

### ✅ Option 2: Promote Your Node to Worker (Advanced)

If you want to keep constraints, you can **label your manager as a worker**:

```bash
docker node update --label-add role=worker $(docker info --format '{{.Swarm.NodeID}}')
```

Then change the constraint to:

```yaml
placement:
  constraints:
    - node.labels.role == worker
```

Or better: Allow **managers to run services**:

```yaml
placement:
  constraints:
    - node.role == worker || node.role == manager
```

👉 This way, on multi-node clusters: workers preferred  
On single-node: manager can run services

## 🧠 Why This Happens

| Fact | Explanation |
|------|-------------|
| You're on a **single-node Swarm** | That node is a **manager** |
| `node.role == worker` | Requires a **worker node** |
| No worker nodes exist | So containers stay in **Pending** state |
| Solution | Allow manager to run services |

---

## ✅ Final Recommendation

For **single-node production or dev setups**, use:

```yaml
placement:
  constraints:
    - node.role == worker || node.role == manager
```

For **multi-node clusters**, keep `node.role == worker` for apps, and run only on managers for control-plane services.
