The phpMyAdmin connection error indicates that MySQL container is likely not starting properly or is rejecting connections due to the new configuration. Here's how to diagnose and fix this:

## Step 1: Check MySQL Container Status

```bash
# Check if MySQL container is running
docker ps

# Check MySQL container logs for errors
docker logs <mysql_container_id>

# Or with docker-compose
docker-compose logs mysql
```

## Step 2: Common Issues with Your New Configuration

Your new configuration has several potential problems:

### Issue 1: Memory Allocation Too High
```ini
innodb_buffer_pool_size=1G
innodb_log_file_size=1G
```
These settings require significant RAM. If your container doesn't have enough memory, MySQL won't start.

### Issue 2: Configuration Syntax
Some settings use different syntax than expected:
```ini
# Wrong
default-authentication-plugin=mysql_native_password

# Correct  
default_authentication_plugin=mysql_native_password
```

## Step 3: Safe Configuration to Start With

Replace your current config with this safer version:

```ini
[mysqld]
# Basic settings
default_authentication_plugin=mysql_native_password

# Import optimizations (safer values)
max_allowed_packet=512M
net_read_timeout=300
net_write_timeout=300
wait_timeout=28800
interactive_timeout=28800

# Conservative performance settings
innodb_buffer_pool_size=256M
innodb_log_buffer_size=64M
innodb_log_file_size=256M
innodb_flush_log_at_trx_commit=2

# Import optimization
bulk_insert_buffer_size=64M

# Connection settings
max_connections=200

[mysql]
default_character_set=utf8mb4

[client]
default_character_set=utf8mb4
```

## Step 4: Restart and Test

```bash
# Restart containers
docker-compose down
docker-compose up -d

# Check MySQL logs
docker-compose logs mysql

# Test connection
docker exec -it <mysql_container> mysql -u laravel_user -p
```

## Step 5: If MySQL Still Won't Start

### Option A: Temporary Minimal Config
```ini
[mysqld]
default_authentication_plugin=mysql_native_password
max_allowed_packet=512M

[client]
default_character_set=utf8mb4
```

### Option B: Check Docker Memory Limits
In your `docker-compose.yml`:
```yaml
services:
  mysql:
    image: mysql:8.0
    deploy:
      resources:
        limits:
          memory: 2G  # Increase if using large buffer pool
        reservations:
          memory: 1G
```

## Step 6: Gradual Configuration Increase

Once MySQL starts successfully:
1. Add one setting at a time
2. Restart and test after each addition
3. Monitor container logs for errors
4. Gradually increase memory settings based on your system's capacity

## Quick Fix Commands

```bash
# 1. Stop everything
docker-compose down

# 2. Update config file with safer settings
nano docker/mysql/my.cnf

# 3. Start with clean slate
docker-compose up -d

# 4. Check if MySQL is ready
docker-compose exec mysql mysqladmin -u root -p ping
```

The issue is likely that your container doesn't have enough resources allocated for the aggressive memory settings (1G buffer pool + 1G log files). Start with the conservative configuration above and gradually increase settings once you confirm MySQL starts properly.
