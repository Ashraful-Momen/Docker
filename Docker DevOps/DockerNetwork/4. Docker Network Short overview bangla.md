# Docker Network সম্পূর্ণ গাইড

## Docker Network কি? (What is Docker Network?)

Docker Network হচ্ছে একটি powerful feature যা বিভিন্ন container গুলোর মধ্যে communication establish করতে ব্যবহৃত হয়। এটি containers গুলোকে একে অপরের সাথে এবং external world এর সাথে যোগাযোগ করতে সাহায্য করে।

## Docker Network এর প্রকারভেদ (Types of Docker Networks)

### 1. **Bridge Network** (ডিফল্ট নেটওয়ার্ক)
- **বাংলা ব্যাখ্যা**: এটি ডিফল্ট network driver। একই host এর containers গুলো এই network এর মাধ্যমে communicate করতে পারে।
- **ব্যবহার**: Single host এ multiple containers run করার জন্য
- **Command**: `docker network create --driver bridge my-bridge`

### 2. **Host Network**
- **বাংলা ব্যাখ্যা**: Container সরাসরি host machine এর network ব্যবহার করে। কোনো network isolation থাকে না।
- **ব্যবহার**: High performance প্রয়োজন হলে
- **Command**: `docker run --network host nginx`

### 3. **Overlay Network**
- **বাংলা ব্যাখ্যা**: Multiple Docker hosts এর মধ্যে containers গুলোকে connect করার জন্য ব্যবহৃত হয়।
- **ব্যবহার**: Docker Swarm mode এ distributed applications এর জন্য
- **Command**: `docker network create --driver overlay my-overlay`

### 4. **Macvlan Network**
- **বাংলা ব্যাখ্যা**: Container গুলোকে physical network এর direct access দেয়।
- **ব্যবহার**: Legacy applications যেগুলো direct network access চায়
- **Command**: `docker network create -d macvlan --subnet=192.168.1.0/24 --gateway=192.168.1.1 -o parent=eth0 my-macvlan`

### 5. **None Network**
- **বাংলা ব্যাখ্যা**: Container এ কোনো network interface থাকে না।
- **ব্যবহার**: Complete network isolation প্রয়োজন হলে
- **Command**: `docker run --network none alpine`

## কেন Docker Network ব্যবহার করবেন? (Why Use Docker Network?)

### 1. **Container Communication (কন্টেইনার যোগাযোগ)**
- বিভিন্ন containers এর মধ্যে secure communication
- Service discovery এবং load balancing
- Database ও application containers এর মধ্যে connection

### 2. **Isolation (বিচ্ছিন্নতা)**
- Network level এ security
- Different environments (dev, staging, prod) আলাদা রাখা
- Resource separation

### 3. **Scalability (স্কেলেবিলিটি)**
- Easy horizontal scaling
- Load distribution
- Multi-host networking

### 4. **Flexibility (নমনীয়তা)**
- Different network topologies
- Custom network configurations
- Integration with existing infrastructure

## Use Cases (ব্যবহারের ক্ষেত্র)

### 1. **Web Application Stack**
```bash
# Database network তৈরি
docker network create app-network

# Database container run
docker run -d --name postgres --network app-network -e POSTGRES_DB=myapp postgres

# Backend API container run
docker run -d --name api --network app-network -p 3000:3000 my-api

# Frontend container run  
docker run -d --name frontend --network app-network -p 80:80 my-frontend
```

### 2. **Microservices Architecture**
```bash
# Microservices network
docker network create microservices-net

# User service
docker run -d --name user-service --network microservices-net user-api

# Order service
docker run -d --name order-service --network microservices-net order-api

# Gateway service
docker run -d --name api-gateway --network microservices-net -p 8080:8080 gateway
```

### 3. **Load Balancer Setup**
```bash
# Load balancer network
docker network create lb-network

# Multiple app instances
docker run -d --name app1 --network lb-network my-app
docker run -d --name app2 --network lb-network my-app
docker run -d --name app3 --network lb-network my-app

# Nginx load balancer
docker run -d --name nginx-lb --network lb-network -p 80:80 nginx-lb-config
```

## কোন Network Type সবচেয়ে ভালো? (Which Network Type is Best?)

### **Bridge Network** - সবচেয়ে জনপ্রিয় ✅
**কখন ব্যবহার করবেন**:
- Single host applications
- Development environment
- Simple container communication

### **Overlay Network** - Production এর জন্য ✅
**কখন ব্যবহার করবেন**:
- Multi-host applications
- Docker Swarm mode
- Production microservices

### **Host Network** - Performance এর জন্য ⚡
**কখন ব্যবহার করবেন**:
- Maximum network performance চাইলে
- Network debugging
- Legacy applications

## Secure Networking Service

### 1. **Network Encryption**
```bash
# Encrypted overlay network
docker network create --driver overlay --opt encrypted secure-network
```

### 2. **Access Control**
```bash
# Custom bridge with specific subnet
docker network create --driver bridge --subnet=172.20.0.0/16 --ip-range=172.20.240.0/20 secure-bridge
```

### 3. **Firewall Rules**
```bash
# Container with specific IP
docker run --network secure-bridge --ip 172.20.240.10 my-app
```

### 4. **Network Policies**
- Docker Compose এ network policies define করা
- External firewall integration
- Service mesh ব্যবহার (Istio, Linkerd)

## Useful Commands (উপযোগী কমান্ড)

### Network Management
```bash
# সব networks দেখুন
docker network ls

# Network details
docker network inspect bridge

# Network তৈরি
docker network create my-network

# Network delete
docker network rm my-network

# Container কে network এ connect
docker network connect my-network my-container

# Container কে network থেকে disconnect  
docker network disconnect my-network my-container
```

### Troubleshooting
```bash
# Container এর network info
docker exec my-container ip addr show

# Network connectivity test
docker exec my-container ping another-container

# Port listening check
docker exec my-container netstat -tulpn
```

## Docker Compose এ Networking

```yaml
version: '3.8'

services:
  frontend:
    build: ./frontend
    networks:
      - frontend-network
    ports:
      - "80:80"

  backend:
    build: ./backend  
    networks:
      - frontend-network
      - backend-network
    ports:
      - "3000:3000"

  database:
    image: postgres
    networks:
      - backend-network
    environment:
      POSTGRES_DB: myapp

networks:
  frontend-network:
    driver: bridge
  backend-network:
    driver: bridge
    internal: true  # External access block
```

## Performance Optimization

### 1. **Network Driver Selection**
- Bridge: General purpose
- Host: Maximum performance
- Overlay: Multi-host with encryption

### 2. **DNS Optimization**
```bash
# Custom DNS

```

### 3. **MTU Configuration**
```bash
# Custom MTU size
docker network create --driver bridge --opt com.docker.network.driver.mtu=1450 my-network
```

## সারমর্ম (Summary)

Docker Network একটি অত্যন্ত শক্তিশালী feature যা:
- **Container Communication** সহজ করে
- **Security** এবং **Isolation** প্রদান করে  
- **Scalability** এবং **Flexibility** বৃদ্ধি করে
- **Production-ready** applications তৈরিতে সাহায্য করে

**সুপারিশ**: নতুনদের জন্য **Bridge Network** দিয়ে শুরু করুন, তারপর প্রয়োজন অনুযায়ী **Overlay Network** ব্যবহার করুন।
