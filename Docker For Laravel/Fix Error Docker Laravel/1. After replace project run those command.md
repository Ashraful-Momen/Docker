The error 500 in Laravel typically indicates a server-side issue. Let's debug this systematically:

## 1. Check Laravel logs first
```bash
docker exec -it laravel_app bash
tail -f storage/logs/laravel.log
```

## 2. Check if the .env file exists and is configured
```bash
# Inside the container
ls -la .env
cat .env
```

If `.env` doesn't exist, copy from example:
```bash
cp .env.example .env
```

## 3. Generate application key
```bash
php artisan key:generate
```

## 4. Install Composer dependencies (if not done)
```bash
composer install --no-dev --optimize-autoloader
```

## 5. Set proper permissions
```bash
# Still inside the container
chown -R www-data:www-data storage bootstrap/cache
chmod -R 775 storage bootstrap/cache
```

## 6. Run database migrations (if needed)
```bash
php artisan migrate
```

## 7. Clear all caches again
```bash
php artisan config:clear
php artisan cache:clear
php artisan route:clear
php artisan view:clear
```

## 8. Check nginx error logs
```bash
# In a new terminal
docker logs laravel_nginx
```

## 9. Check PHP-FPM logs
```bash
docker logs laravel_app
```

## 10. Debug mode (temporary)
Edit your `.env` file:
```bash
APP_DEBUG=true
APP_ENV=local
```

Then clear config:
```bash
php artisan config:clear
```

## 11. Test if PHP is working
Create a simple PHP info file:
```bash
echo "<?php phpinfo(); ?>" > public/info.php
```

Then visit `http://localhost:8080/info.php` to see if PHP is working.

**Most common causes of Error 500:**
- Missing `.env` file
- No application key (`APP_KEY` not set)
- Permission issues with `storage/` and `bootstrap/cache/`
- Missing Composer dependencies
- Database connection issues

Start with checking the Laravel logs - they'll tell you exactly what's wrong. What do you see in `storage/logs/laravel.log`?
