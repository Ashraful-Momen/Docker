# Laravel Docker CI/CD Optimization Guide

## 1. Optimized Multi-Stage Dockerfile

Create a new optimized `Dockerfile`:

```dockerfile
# Multi-stage build for Laravel application
# Stage 1: Composer dependencies
FROM composer:2 AS composer
WORKDIR /app
COPY composer.json composer.lock ./
RUN composer install --no-dev --optimize-autoloader --no-interaction --no-scripts --ignore-platform-reqs

# Stage 2: Node.js assets build
FROM node:18-alpine AS node
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
RUN npm run build

# Stage 3: Final production image
FROM php:8.3-fpm-alpine AS production

# Install only essential system dependencies
RUN apk add --no-cache \
    freetype-dev \
    libjpeg-turbo-dev \
    libpng-dev \
    libzip-dev \
    icu-dev \
    oniguruma-dev \
    mysql-client \
    nginx \
    supervisor

# Install PHP extensions (minimal set)
RUN docker-php-ext-configure gd --with-freetype --with-jpeg \
    && docker-php-ext-install -j$(nproc) \
        pdo pdo_mysql mbstring zip gd \
        intl opcache bcmath

# Install Redis extension
RUN apk add --no-cache --virtual .build-deps $PHPIZE_DEPS \
    && pecl install redis \
    && docker-php-ext-enable redis \
    && apk del .build-deps

# Create application user
RUN addgroup -g 1000 www && adduser -u 1000 -G www -s /bin/sh -D www

# Set working directory
WORKDIR /var/www/html

# Copy composer dependencies
COPY --from=composer /app/vendor ./vendor

# Copy built assets
COPY --from=node /app/public ./public

# Copy application files
COPY --chown=www:www . .

# Copy configuration files
COPY ./docker/php/php.ini /usr/local/etc/php/conf.d/app.ini
COPY ./docker/nginx/nginx.conf /etc/nginx/nginx.conf
COPY ./docker/supervisord.conf /etc/supervisor/conf.d/supervisord.conf

# Set permissions
RUN mkdir -p /var/www/html/storage/logs \
    /var/www/html/storage/framework/cache \
    /var/www/html/storage/framework/sessions \
    /var/www/html/storage/framework/views \
    /var/www/html/bootstrap/cache \
    && chown -R www:www /var/www/html \
    && chmod -R 775 /var/www/html/storage \
    && chmod -R 775 /var/www/html/bootstrap/cache

# Laravel optimizations
RUN php artisan config:cache \
    && php artisan route:cache \
    && php artisan view:cache

USER www

EXPOSE 80

CMD ["/usr/bin/supervisord", "-c", "/etc/supervisor/conf.d/supervisord.conf"]
```

```ini
opcache.max_wasted_percentage=10

```

### docker/nginx/nginx.conf
```nginx
user www;
worker_processes auto;
pid /var/run/nginx.pid;

events {
    worker_connections 1024;
    use epoll;
    multi_accept on;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;
    
    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    
    gzip on;
    gzip_vary on;
    gzip_types text/css application/javascript text/javascript application/json;
    
    server {
        listen 80;
        server_name localhost;
        root /var/www/html/public;
        index index.php;
        
        location / {
            try_files $uri $uri/ /index.php?$query_string;
        }
        
        location ~ \.php$ {
            fastcgi_pass 127.0.0.1:9000;
            fastcgi_index index.php;
            fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
            include fastcgi_params;
        }
        
        location ~ /\.ht {
            deny all;
        }
    }
}
```

## 3. Production Docker Compose

### docker-compose.prod.yml
```yaml
version: '3.8'

services:
  app:
    build:
      context: .
      dockerfile: Dockerfile
      target: production
    restart: unless-stopped
    environment:
      - APP_ENV=production
      - APP_DEBUG=false
      - DB_HOST=mysql
      - REDIS_HOST=redis
      - CACHE_DRIVER=redis
      - SESSION_DRIVER=redis
    networks:
      - laravel_network
    depends_on:
      - mysql
      - redis

  mysql:
    image: mysql:8.0
    restart: unless-stopped
    environment:
      MYSQL_DATABASE: ${DB_DATABASE}
      MYSQL_ROOT_PASSWORD: ${DB_ROOT_PASSWORD}
      MYSQL_USER: ${DB_USERNAME}
      MYSQL_PASSWORD: ${DB_PASSWORD}
    volumes:
      - mysql_data:/var/lib/mysql
    networks:
      - laravel_network
    command: >
      --default-authentication-plugin=mysql_native_password
      --innodb-buffer-pool-size=1G
      --innodb-log-file-size=256M

  redis:
    image: redis:7-alpine
    restart: unless-stopped
    networks:
      - laravel_network
    command: redis-server --appendonly yes

volumes:
  mysql_data:

networks:
  laravel_network:
    driver: bridge
```

## 4. GitLab CI/CD Pipeline

### .gitlab-ci.yml
```yaml
stages:
  - test
  - build
  - deploy

variables:
  DOCKER_REGISTRY: $CI_REGISTRY
  IMAGE_NAME: $CI_REGISTRY_IMAGE
  DOCKER_DRIVER: overlay2
  DOCKER_TLS_CERTDIR: "/certs"

services:
  - docker:24-dind

before_script:
  - docker login -u $CI_REGISTRY_USER -p $CI_REGISTRY_PASSWORD $CI_REGISTRY

# Test stage
test:
  stage: test
  image: php:8.3-alpine
  services:
    - mysql:8.0
  variables:
    MYSQL_DATABASE: laravel_test
    MYSQL_ROOT_PASSWORD: test_password
    DB_HOST: mysql
    DB_DATABASE: laravel_test
    DB_USERNAME: root
    DB_PASSWORD: test_password
  before_script:
    - apk add --no-cache git zip unzip
    - curl -sS https://getcomposer.org/installer | php -- --install-dir=/usr/local/bin --filename=composer
    - docker-php-ext-install pdo pdo_mysql
  script:
    - composer install --prefer-dist --no-ansi --no-interaction --no-progress --no-scripts
    - cp .env.testing .env
    - php artisan key:generate
    - php artisan migrate --force
    - php artisan test
  only:
    - merge_requests
    - main

# Build stage
build:
  stage: build
  image: docker:24
  script:
    - |
      if [[ "$CI_COMMIT_BRANCH" == "main" ]]; then
        TAG="latest"
      else
        TAG="$CI_COMMIT_SHORT_SHA"
      fi
    - echo "Building image with tag: $TAG"
    - docker build --pull -t $IMAGE_NAME:$TAG .
    - docker push $IMAGE_NAME:$TAG
    - docker tag $IMAGE_NAME:$TAG $IMAGE_NAME:$CI_COMMIT_SHORT_SHA
    - docker push $IMAGE_NAME:$CI_COMMIT_SHORT_SHA
  only:
    - main
    - develop

# Deploy to staging
deploy_staging:
  stage: deploy
  image: alpine:latest
  before_script:
    - apk add --no-cache openssh-client
    - eval $(ssh-agent -s)
    - echo "$STAGING_PRIVATE_KEY" | tr -d '\r' | ssh-add -
    - mkdir -p ~/.ssh
    - chmod 700 ~/.ssh
    - ssh-keyscan $STAGING_HOST >> ~/.ssh/known_hosts
    - chmod 644 ~/.ssh/known_hosts
  script:
    - |
      ssh $STAGING_USER@$STAGING_HOST << 'ENDSSH'
        cd /var/www/laravel-app
        docker login -u $CI_REGISTRY_USER -p $CI_REGISTRY_PASSWORD $CI_REGISTRY
        docker pull $IMAGE_NAME:$CI_COMMIT_SHORT_SHA
        docker tag $IMAGE_NAME:$CI_COMMIT_SHORT_SHA laravel_app:latest
        docker-compose -f docker-compose.prod.yml up -d app
        docker image prune -f
      ENDSSH
  environment:
    name: staging
    url: https://staging.yourapp.com
  only:
    - develop

# Deploy to production
deploy_production:
  stage: deploy
  image: alpine:latest
  before_script:
    - apk add --no-cache openssh-client
    - eval $(ssh-agent -s)
    - echo "$PRODUCTION_PRIVATE_KEY" | tr -d '\r' | ssh-add -
    - mkdir -p ~/.ssh
    - chmod 700 ~/.ssh
    - ssh-keyscan $PRODUCTION_HOST >> ~/.ssh/known_hosts
    - chmod 644 ~/.ssh/known_hosts
  script:
    - |
      ssh $PRODUCTION_USER@$PRODUCTION_HOST << 'ENDSSH'
        cd /var/www/laravel-app
        docker login -u $CI_REGISTRY_USER -p $CI_REGISTRY_PASSWORD $CI_REGISTRY
        docker pull $IMAGE_NAME:latest
        docker tag $IMAGE_NAME:latest laravel_app:latest
        docker-compose -f docker-compose.prod.yml up -d app
        docker exec laravel_app_1 php artisan migrate --force
        docker exec laravel_app_1 php artisan cache:clear
        docker image prune -f
      ENDSSH
  environment:
    name: production
    url: https://yourapp.com
  when: manual
  only:
    - main

# Cleanup old images
cleanup:
  stage: deploy
  image: docker:24
  script:
    - docker system prune -af
    - docker image prune -af --filter "until=72h"
  when: manual
  allow_failure: true
```

## 6. Deployment Workflow

### Step-by-Step Process:

1. **Initial Setup**:
   ```bash
   # Clone your repository on the production server
   git clone https://gitlab.com/username/your-laravel-app.git
   cd your-laravel-app
   
   # Create production environment file
   cp .env.production .env
   
   # Edit the .env file with your production values
   nano .env
   ```

2. **GitLab CI/CD Variables Setup**:
   - Go to GitLab → Settings → CI/CD → Variables
   - Add the following variables:
     ```
     STAGING_HOST: your-staging-server-ip
     STAGING_USER: deploy
     STAGING_PRIVATE_KEY: your-ssh-private-key
     PRODUCTION_HOST: your-production-server-ip
     PRODUCTION_USER: deploy
     PRODUCTION_PRIVATE_KEY: your-ssh-private-key
     ```

3. **Server Preparation**:
   ```bash
   # Install Docker and Docker Compose on your servers
   curl -fsSL https://get.docker.com -o get-docker.sh
   sh get-docker.sh
   
   # Add deploy user to docker group
   sudo usermod -aG docker deploy
   
   # Install Docker Compose
   sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
   sudo chmod +x /usr/local/bin/docker-compose
   ```

4. **First Deployment**:
   ```bash
   # On your production server
   cd /var/www/your-laravel-app
   docker-compose -f docker-compose.prod.yml up -d
   
   # Run Laravel setup commands
   docker exec laravel_app_1 php artisan key:generate
   docker exec laravel_app_1 php artisan migrate --seed
   docker exec laravel_app_1 php artisan storage:link
   ```

## 7. Optimizations Implemented

### Docker Image Size Reduction:
- **Multi-stage builds**: Separate build and runtime environments
- **Alpine base images**: Smaller footprint (~50MB vs 200MB+)
- **Layer optimization**: Better caching and fewer layers
- **Build-time dependencies**: Removed after build
- **Expected final size**: ~200-300MB (vs your current 4.6GB)

### CI/CD Optimizations:
- **Parallel builds**: Test and build stages run independently
- **Docker layer caching**: Faster subsequent builds
- **Registry cleanup**: Automatic old image removal
- **Environment-specific deployments**: Staging/Production separation
- **Manual production deployment**: Safety gate for production

### Performance Optimizations:
- **OPcache enabled**: PHP bytecode caching
- **Laravel caching**: Config, route, and view caching
- **Nginx + PHP-FPM**: Better resource utilization
- **Supervisor**: Process management for multiple services

## 9. Security Considerations

1. **Use secrets management** for sensitive data
2. **Regular image updates** via scheduled pipelines
3. **Vulnerability scanning** in CI/CD
4. **Non-root containers** (implemented in Dockerfile)
5. **Network isolation** between services

This setup should reduce your image size by ~90% and provide a robust, production-ready CI/CD pipeline for your Laravel application.
