Perfect 👍 You already added a small override for `uploads.ini`, but for **really big database imports**, phpMyAdmin needs higher limits on both **PHP** and **Nginx/Apache inside the container**.

Here’s how to update it:

---

### 1. Update `docker-compose.yml` → phpMyAdmin service

```yaml
  phpmyadmin:
    image: phpmyadmin/phpmyadmin
    container_name: laravel_phpmyadmin
    restart: unless-stopped
    environment:
      PMA_HOST: mysql
      PMA_PORT: 3306
      PMA_USER: root
      PMA_PASSWORD: root_password
      UPLOAD_LIMIT: 4096M      # allow up to 4GB uploads
      MEMORY_LIMIT: 4096M      # PHP memory
      MAX_EXECUTION_TIME: 0    # unlimited import time
      MAX_INPUT_TIME: -1
    ports:
      - "8081:80"
    volumes:
      - ./docker/phpmyadmin/uploads.ini:/usr/local/etc/php/conf.d/uploads.ini
    networks:
      - laravel_network
    depends_on:
      - mysql
```

---

### 2. Create `./docker/phpmyadmin/uploads.ini`

```ini
; PHP settings for phpMyAdmin imports
upload_max_filesize = 4096M
post_max_size = 4096M
memory_limit = 4096M
max_execution_time = 0
max_input_time = -1
```

---

### 3. Apply changes

```bash
docker-compose down
docker-compose up -d
```

---

✅ Now phpMyAdmin can handle **multi-GB `.sql` imports**.

⚡ But note: importing **very large SQL files via browser** is still slow and unreliable.
👉 For really big databases, it’s better to import directly into MySQL with:

```bash
docker exec -i laravel_mysql mysql -u root -proot_password laravel_db < /path/to/huge.sql
```
