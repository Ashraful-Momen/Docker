I see exactly why your **build is slow** and the **image is huge** 🛑

Let’s break it down:

---

### 🔎 Problems in your Dockerfile

1. **`apt-get install` installs a lot of heavy packages**

   * `build-essential` (\~150MB)
   * `default-mysql-client` (\~100MB)
   * `git`, `curl`, etc. all add up
   * You don’t remove unnecessary packages after building extensions.

2. **`COPY . /var/www/html` before permissions**

   * You copy the **entire Laravel project**, including `vendor/` and `node_modules/`.
   * These folders alone can add **hundreds of MB** to your image.
   * Then you run `chown -R` and `chmod -R`, which takes forever because it’s recursive on thousands of files.

3. **Permissions step is very expensive**

   ```dockerfile
   RUN chown -R www:www /var/www/html
   ```

   * This traverses your whole Laravel project every build.
   * Since you’re mounting with volumes anyway, it’s wasted work.

4. **Final image has dev dependencies**
   You’re keeping compilers (`build-essential`, `pkg-config`, headers) inside the runtime image, which bloats it.

---

### ⚡ How to Fix (Smaller & Faster)

#### ✅ Use multi-stage build

One stage for building PHP extensions and installing composer deps, another for running.

Example optimized Dockerfile:

```dockerfile
# Stage 1: Base PHP with extensions
FROM php:8.3-fpm AS base

WORKDIR /var/www/html

# Install only what’s needed for extensions
RUN apt-get update && apt-get install -y \
    libpng-dev libjpeg62-turbo-dev libfreetype6-dev \
    libzip-dev libicu-dev libonig-dev libxml2-dev \
    libpq-dev libssl-dev \
    zip unzip git curl \
    && docker-php-ext-configure gd --with-freetype --with-jpeg \
    && docker-php-ext-install -j$(nproc) \
        pdo pdo_mysql pdo_pgsql mbstring zip exif pcntl gd xml bcmath soap intl mysqli opcache \
    && pecl install redis \
    && docker-php-ext-enable redis \
    && apt-get purge -y --auto-remove \
    && rm -rf /var/lib/apt/lists/*

COPY --from=composer:latest /usr/bin/composer /usr/bin/composer

COPY ./docker/php/local.ini /usr/local/etc/php/conf.d/local.ini

# Stage 2: Final runtime
FROM php:8.3-fpm

WORKDIR /var/www/html

# Copy built extensions from base
COPY --from=base /usr/local/lib/php/extensions /usr/local/lib/php/extensions
COPY --from=base /usr/local/etc/php/conf.d /usr/local/etc/php/conf.d
COPY --from=base /usr/bin/composer /usr/bin/composer

# Create user
RUN groupadd -g 1000 www \
    && useradd -u 1000 -ms /bin/bash -g www www

# Copy only Laravel source (without vendor/node_modules)
COPY . /var/www/html
RUN rm -rf /var/www/html/vendor /var/www/html/node_modules

# Fix only writable dirs (no full chown)
RUN mkdir -p /var/www/html/storage /var/www/html/bootstrap/cache \
    && chown -R www:www /var/www/html/storage /var/www/html/bootstrap/cache

USER www

EXPOSE 9000
CMD ["php-fpm"]
```

---

### 🚀 Why this is better

* **Image size reduced**: Drops compilers & dev packages in final stage.
* **Build speed faster**: `chown` only runs on `storage/` and `bootstrap/cache`, not the whole project.
* **No bloat from vendor/node\_modules**: They are mounted as volumes or installed in CI/CD pipeline.
* **Layer caching works better**: Dependencies installed once, not invalidated by source code changes.
