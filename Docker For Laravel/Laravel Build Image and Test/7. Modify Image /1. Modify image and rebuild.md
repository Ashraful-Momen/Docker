# Quick Laravel Docker Image Rebuild Process

## Method 1: Layer-Based Approach (Fastest - ~30 seconds)

### Step 1: Pull the existing image
```bash
docker pull yourusername/your-laravel-app:latest
```

### Step 2: Create a new Dockerfile for env changes
```dockerfile
# Dockerfile.env
FROM yourusername/your-laravel-app:latest

# Copy the new .env file
COPY .env /var/www/html/.env

# If you need to clear config cache
RUN php artisan config:clear && php artisan config:cache
```

### Step 3: Prepare your new .env file
Create your updated `.env` file in the current directory with your server-specific configurations.

### Step 4: Build the new image
```bash
docker build -f Dockerfile.env -t yourusername/your-laravel-app:updated .
```

### Step 5: Run the updated container
```bash
docker run -d -p 80:80 yourusername/your-laravel-app:updated
```

---

## Method 2: Volume Mount Approach (Even Faster - No rebuild needed)

### Step 1: Pull the image
```bash
docker pull yourusername/your-laravel-app:latest
```

### Step 2: Create .env file on host
```bash
# Create your .env file locally
nano /path/to/your/.env
```

### Step 3: Run with volume mount
```bash
docker run -d \
  -p 80:80 \
  -v /path/to/your/.env:/var/www/html/.env \
  yourusername/your-laravel-app:latest
```

---

## Method 3: Multi-stage Build Optimization (For permanent changes)

If you need to push the updated image back to Docker Hub:

### Step 1: Create optimized Dockerfile
```dockerfile
# Dockerfile.optimized
FROM yourusername/your-laravel-app:latest as base

# Copy new environment file
COPY .env /var/www/html/.env

# Clear and cache config
RUN php artisan config:clear && \
    php artisan config:cache && \
    php artisan route:cache && \
    php artisan view:cache

# Final stage
FROM base
WORKDIR /var/www/html
```

### Step 2: Build with BuildKit for faster builds
```bash
DOCKER_BUILDKIT=1 docker build \
  -f Dockerfile.optimized \
  -t yourusername/your-laravel-app:v2 .
```

### Step 3: Push updated image
```bash
docker push yourusername/your-laravel-app:v2
```

## Speed Optimization Tips

### 1. Use .dockerignore
```bash
# .dockerignore
.git
.gitignore
README.md
.env.example
node_modules
storage/logs/*
```

### 2. Enable BuildKit
```bash
export DOCKER_BUILDKIT=1
```

### 3: Use build cache
```bash
docker build --cache-from yourusername/your-laravel-app:latest \
  -t yourusername/your-laravel-app:updated .
```

### 4: Parallel builds (if multiple changes)
```bash
docker buildx create --use
docker buildx build --platform linux/amd64 -t your-image:tag .
```

---

## Recommended Approach

**For quickest deployment:** Use **Method 2 (Volume Mount)** - No rebuild time, instant deployment.

**For production with permanent changes:** Use **Method 1 (Layer-Based)** - Minimal rebuild time (~30 seconds).

**For pushing to registry:** Use **Method 3 (Multi-stage)** - Optimized for distribution.
