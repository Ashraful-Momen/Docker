# 🧠 Final Architecture

```text
Internet
   ↓
Nginx (Docker, port 80/443)
   ├── /          → Next.js (PM2, localhost:3000)
   └── /uploads   → /var/www/uploads (host disk)
   
Docker Compose:
   ├── PostgreSQL
   ├── pgAdmin
   └── Redis

Next.js:
   └── Running via PM2 (outside Docker)
```

👉 This is **hybrid architecture (very common in production)**

---

# ✅ 1. Create upload directory (host machine)

```bash
sudo mkdir -p /var/www/uploads
sudo chown -R $USER:$USER /var/www/uploads
sudo chmod -R 755 /var/www/uploads
```

---

# ✅ 2. Update your Next.js upload path

In your API (multer or whatever):

```js
const uploadPath = "/var/www/uploads";
```

Return URL like:

```js
fileUrl: `/uploads/${filename}`
```

# ✅ 3. Add Nginx to Docker Compose

Update your `docker-compose.yml`:

```yaml
version: '3.8'

services:
  nginx:
    image: nginx:latest
    container_name: nginx_server
    restart: unless-stopped
    ports:
      - "80:80"
    volumes:
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
      - /var/www/uploads:/var/www/uploads
    depends_on:
      - postgres
      - redis

  postgres:
    image: postgres:15-alpine
    container_name: postgres_db
    restart: unless-stopped
    environment:
      POSTGRES_USER: admin
      POSTGRES_PASSWORD: StrongPassword123!
      POSTGRES_DB: mydatabase
    volumes:
      - postgres_data:/var/lib/postgresql/data

  pgadmin:
    image: dpage/pgadmin4
    container_name: pgadmin
    restart: unless-stopped
    ports:
      - "5050:80"

  redis:
    image: redis:alpine
    container_name: redis_cache
    restart: unless-stopped

volumes:
  postgres_data:
```

# ✅ 4. Create `nginx.conf` (IMPORTANT)

```nginx
server {
    listen 80;
    server_name your-domain.com;  # or your VPS IP

    # ✅ Static uploads (VERY FAST)
    location /uploads/ {
        alias /var/www/uploads/;
        autoindex off;

        expires 30d;
        add_header Cache-Control "public, no-transform";

        # security
        location ~* \.(php|js|sh|exe)$ {
            deny all;
        }
    }

    # ✅ Next.js app
    location / {
        proxy_pass http://host.docker.internal:3000;

        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';

        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

---

## ⚠️ Important note (Linux users)

`host.docker.internal` may not work by default.

### Fix:

Use your host IP (example):

```nginx
proxy_pass http://172.17.0.1:3000;
```

OR run nginx with host network:

```yaml
network_mode: "host"
```

# ✅ 6. Run Next.js with PM2 (outside Docker)

Install PM2:

```bash
npm install -g pm2
```

Build app:

```bash
npm run build
```

Run with PM2:

```bash
pm2 start npm --name "nextjs-app" -- start
```

Save process:

```bash
pm2 save
pm2 startup
```

# ⚠️ Real-world issues you might face

### 1. ❌ Permission denied

Fix:

```bash
chmod -R 755 /var/www/uploads
```

---

### 2. ❌ Images not loading

* Wrong `alias` path
* Missing trailing `/`

---

### 3. ❌ Nginx can't reach Next.js

* Wrong IP (`host.docker.internal`)
* Fix with `172.17.0.1`

---

### 4. ❌ PM2 not restarting on reboot

```bash
pm2 startup
pm2 save
```
